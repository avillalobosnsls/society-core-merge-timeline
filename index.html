How design & content requests flow

The working agreement between Product and Marketing

  • Planned by default — requests tied to a release go on a shared timeline with the build schedule, so nothing lands as a surprise and nothing gets cut for time
  • Ad-hoc lane exists — content materials sometimes can't wait for a planning cycle; flag them as ad-hoc and we triage together in this sync (the exception, not the default)
  • This sync is the venue — planned items get slotted, ad-hoc items get triaged, and this timeline stays current
❓ Question for the group: what lead time does the team realistically need for release-tied content (e.g., the v1.5 launch campaign)? That number becomes the anchor for the request timeline.
✍️ And one thing we can start now: customized Society banner copy for each Members Area entry point (post-induction, Orientation, LTD, SNTs) — v1.5 is paused, so this is pressure-free runway.
